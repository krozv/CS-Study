## 1. 시스템 소프트웨어와 응용 소프트웨어의 차이는?
    시스템 소프트웨어는 컴퓨터를 동작시키기 위한 명령어 집합이고 응용 소프트웨어는 특수한 목적을 달성하기 위한 명령어 집합입니다.
## 2. 시스템 버스란?
    시스템 버스는 컴퓨터의 각 부품들이 통신하기 위한 통로입니다. 시스템 버스는 기본적으로 데이터 버스, 주소 버스, 제어 버스가 존재합니다. 데이터 버스는 데이터를 전달하기 위한 양방향 버스입니다. 입출력장치와 메모리에서 데이터를 CPU로 보내거나 CPU에서 입출력장치와 메모리로 데이터를 보냅니다. 주소 버스는 데이터가 전달될 주소를 보내는 단방향 버스입니다. CPU에서 입출력장치와 메모리로 데이터가 저장될 주소를 보내는 역할을 합니다. 마지막으로 제어 버스는 CPU에서 입출력장치나 메모리로 제어 신호를 보내는 양방향 버스입니다. 제어 신호의 종류로는 기억장치 읽기/쓰기, 버스 요청 및 승인, 인터럽트 요청/승인, 클락, 리셋 등이 있습니다.
## 3. 명령어 처리 과정은?
    CPU 내부의 명령어 처리 과정은 다음과 같습니다. 프로그램 카운터에 다음 인출될 명령어의 주소가 저장되어 있어 이를 인출해 메모리 주소 레지스터에 저장합니다. 메모리 주소 레지스터의 주소를 기반으로 메모리로부터 명령어를 읽어 메모리 버퍼 레지스터에 저장하고 프로그램 카운터를 1 증가시킵니다. 이제 메모리 버퍼 레지스터에 저장된 명령어를 명령어 레지스터에 저장한 뒤, 제어장치가 명령어 레지스터의 명령어를 해석하고 제어 신호를 발생시킵니다. 제어 신호는 ALU로 보내져 명령이 수행됩니다.
## 4. 고정 소수점과 부동 소수점은?
    둘 다 컴퓨터에서 실수를 표현하기 위한 방식입니다. 정수부와 소수부를 이진수로 변환한 뒤 어떤 식으로 저장하느냐에 따라서 구분이 되는데, 고정 소수점 방식은 각각 이진수로 변환한 값을 정수부와 소수부에 그대로 저장하는 방식이고, 부동 소수점 방식은 변환한 값을 1.xx * 2^n 꼴로 정규화해서 저장하는 방식입니다. 소수점이 이동하기 때문에 부동 소수점 방식이라고 합니다. 
## 5. RISC, CISC는?
    RISC와 CISC는 CPU 설계에 관한 두 가지 다른 철학입니다. RISC는 Reduced Instruction Set Computer로 CPU 명령어의 개수를 줄여 해석시간을 줄임으로써 개별 명령어의 실행속도를 빠르게 하는 컴퓨터이며, CISC는 Complex Instruction Set Computer로 복잡한 명령어 집합을 갖는 컴퓨터입니다. 명령어가 복잡하기 때문에 시간이 오래 걸리며 해석에 필요한 회로도 복잡합니다.
## 6. 파이프라이닝이란?
    명령어의 인출과 해석, 실행, 저장이 병렬적으로 이루어지는 기술입니다. 각 단계가 겹치지 않는다면 여러 명령을 동시에 실행할 수 있기 때문에 속도가 빨라질 수 있습니다. 하나의 코어에 여러 개의 스레드를 실행하는 방식입니다.
## 7. 캐시메모리란?
    메인 메모리와 CPU 간의 데이터 교환 속도 향상을 위한 중간 버퍼 역할을 하는 메모리입니다. 데이터 지역성을 활용해 메인 메모리에 있는 데이터를 캐시 메모리에 불러와 두고 필요한 데이터를 캐시 메모리에서 먼저 찾도록 하면 시스템 성능을 향상시킬 수 있습니다.
## 8. HDD와 SSD란?
    2차 메모리 즉 보조 기억장치로 메인 메모리에 비해 느리지만 대용량입니다. HDD는 Hard Disk Drive로 플래터와 각 플래터에 대해 읽고 쓸 수 있는 액추에이터 암, 그리고 플래터를 회전시키고 암을 구동하는 모터로 구성되어 있습니다. 각 플래터는 트랙이라고 하는 세부 단위로 구성되며 트랙은 다시 섹터 단위로 나뉘어집니다. SSD는 Solid State Drive로 플래시 메모리를 사용합니다. 플래시 메모리는 셀로 구성되어 있고 셀에 전자를 저장해 이를 기준으로 전류를 통제하므로써 디지털 신호를 표현합니다. 
## 9. CPU란?
    CPU는 컴퓨터의 두뇌역할을 하는 부품으로 기본적으로 ALU, 제어장치, 레지스터로 구성됩니다. ALU는 산술논리장치로 명령어와 제어신호를 받아들여 연산을 수행하는 장치입니다. 제어장치는 레지스터의 정보와 인출된 명령어를 해석해서 제어 신호를 생성하는 장치입니다. 레지스터는 CPU 내에서 필요한 정보를 저장하는 장치들입니다.
## 10. GPU란?
    GPU는 부동 소수점 연산 등의 특화된 연산을 처리하기 위해 제어장치와 캐시를 공유하고 연산을 위한 ALU의 양을 극단적으로 끌어올린 연산장치입니다. 한번에 하나의 연산만 가능한 CPU와는 달리 GPU는 다중 코어 아키텍쳐 덕분에 여러 작업을 병렬로 처리할 수 있습니다.
## 11. 명령어의 구조? (연산 코드, 오프랜드)
    명령어는 명령어가 수행할 연산에 대해 나타내는 명령코드 Operation Code와 연산에 사용할 데이터나 데이터가 저장된 위치를 나타내는 오퍼랜드로 구성되어 있습니다. 명령코드의 종류는 다양하지만, 크게 데이터를 전송하는 MOVE, STORE, LOAD, PUSH, POP, 산술/논리 연산을 하는 ADD, SUBTRACT, MULTIPLY, DIVIDE, 제어의 흐름을 변경하는 JUMP, CONDITIONAL JUMP, HALT, CALL, RETURN, 입출력을 제어하는 READ, WRITE 등이 있습니다. 오퍼랜드는 다른 말로 주소 필드라고 하는데, 명령어의 길이가 대개 긴 경우가 많기 때문에 데이터를 직접 명시하기 보다는 연산에 사용할 데이터가 저장된 메모리 주소나 레지스터 이름을 담기 때문입니다. 
## 12. 슈퍼스칼라란?
    파이프라인을 처리할 수 있는 코어를 여러 개 구성해 복수의 명령어가 동시에 실행되도록 하는 방식입니다. 대부분 파이프라인 기법과 비슷하지만, 각 단계에서 실행되는 명령어의 개수가 더 많으므로 더 빠르게 처리할 수 있습니다. 오늘날의 CPU는 대부분 슈퍼스칼라 기법을 사용합니다.

### 슈퍼스칼라 아키텍처의 장점은?
    한 클록 사이클 당 처리할 수 있는 명령어의 수가 크게 증가합니다. 또 여러 실행 유닛을 동시에 사용해 하드웨어 자원의 유휴 시간을 줄일 수 있습니다. 

## 13. 저장 장치 계층 구조란?
    메모리 계층 구조 즉 Memory Hierarchy라고도 하는데요. 메모리 관련 세 가지 주요 특성인 용량, 접근 속도, 비용 간의 관계를 파악해 필요에 따라 채택할 수 있게 만들어 놓은 구조입니다. 속도가 가장 빠른 장치 순으로 레지스터, 캐시, 메모리, 2차 저장장치 순입니다. 그리고 그 역순으로 용량이 큽니다.

### 레지스터와 캐시 메모리의 차이점은?
    레지스터와 캐시 메모리 둘 다 CPU의 연산을 위해 명령어와 데이터를 저장하는 공간입니다. 레지스터와 캐시 메모리 둘 다 CPU 내부에 위치하지만 레지스터는 연산만을 위한 CPU의 필수 구성 요소지만 캐시 메모리는 저장을 위한 CPU의 보조 구성 요소입니다.

## 14. 컴파일러와 인터프리터?
    컴파일러와 인터프리터 둘 다 사용자가 작성한 고급 프로그래밍 언어를 어셈블리어로 번역해주는 방식입니다. 컴파일러는 프로그램 전체를 스캔해서 한번에 기계가 이해할 수 있는 오브젝트 코드로 번역하고 다시 오브젝트 코드들을 링킹해서 실행파일로 만듭니다. 따라서 컴파일러는 실제 프로그램이 실행되는 데까지 시간이 걸리지만, 한번 실행파일로 만들어 놓으면 빠르게 실행할 수 있습니다. 반면 인터프리터 방식은 코드를 한번에 한 문장씩 번역합니다. 따라서 실행 자체는 빠르게 실행될 수 있지만, 프로그램의 오류가 실행 시점에 파악되므로 오류가 발생하면 바로 중단될 수 있습니다.

## 15. 주기억장치와 보조기억장치의 차이점은?
    주기억장치에는 RAM과 ROM이 있지만, BIOS에 주로 쓰이는 비휘발성 메모리 ROM은 제외하고 RAM를 주기억장치로 상정하고 설명드리겠습니다. 두 기억장치의 주 차이점은 전원이 끊어졌을 때 데이터가 사라지는지 여부입니다. RAM은 데이터를 읽는 속도가 보조기억장치보다 빠르지만 전원이 끊어지면 전자가 유지되지 않고 휘발되므로 데이터가 사라집니다. 그러므로 영구적으로 저장해야 하는 데이터가 있을 경우 보조기억장치에 저장해야합니다. 보조기억장치는 RAM보다는 읽는 속도가 느리지만, 디스크나 반도체에 반영구적으로 저장할 수 있습니다.

## 16. RAM과 HDD의 차이는?
    RAM은 주기억장치, HDD는 보조기억장치입니다.
### 입출력장치의 역할은?
    입출력장치는 컴퓨터의 핵심 요소인 CPU에 저장된 데이터를 보내거나 CPU가 연산한 결과를 저장하는 하드웨어 장치입니다. 또, 마우스나 키보드 등의 입출력장치는 사용자의 입력을 컴퓨터로 보내고 모니터처럼 컴퓨터 내부의 정보를 사용자에게 보여줄 수도 있습니다.

## 17. 입출력장치는 주기억장치와 어떻게 연결되나요?
    입출력장치는 메인보드에 존재하는 장치 컨트롤러라는 하드웨어을 통해 연결됩니다. 장치 컨트롤러는 각 입출력장치를 규격화해서 CPU와 통신을 중개합니다. 서로 간의 데이터 전달은 시스템 버스를 통해서 전달됩니다.
### 메모리 버스와 PCIe 버스의 차이점은?
    메모리 버스는 CPU와 RAM을 연결하는 버스이며, PCIe 버스는 CPU와 네트워크 카드, SSD, GPU와 같은 주변 장치를 연결하는 버스입니다.
## 18. 캐시 일관성이란?
    캐시 일관성이란 두 개 이상의 코어를 가진 멀티 코어 시스템에서 CPU의 캐시 데이터와 메인 공유 메모리의 데이터가 일치되도록 하는 것입니다. 한 코어가 메모리의 내용을 갱신하는 경우 이 사실을 다른 코어가 인지할 수 있도록 하고 기존 데이터를 무효화하는 과정을 구현한 것입니다.

## 19. 파이프라인 해저드란?
    파이프라인 해저드는 파이프라이닝에서 어떤 원인에 의해 다음 클럭에 명령어가 실행되지 않는 현상입니다. 원인으로는 크게 구조적 해저드, 데이터 해저드, 컨트롤 해저드가 있습니다. 구조적 해저드는 구조적으로 자원은 하나인데 여러 명령이 동시에 실행되려고 할 때 발생합니다. 데이터 해저드는 아직 명령이 끝나지 않은 레지스터에 접근해 명령어를 실행하려고 할 때 종속 문제 등으로 발생합니다. 그리고 제어 해저드는 JUMP, BRANCH 등의 분기 명령어가 실행된 시점에 수행되지 않는 명령들이 파이프라인에 존재할 경우 발생합니다.

### 파이프라인 해저드를 해결하는 방법은?
    위험이 해결될 때까지 파이프라인 단계를 지연시키는 Stall, 지연을 피하기 위해 이후 단계에서 이전 단계로 데이터를 라우팅시키는 Forward, 파이프라인 분기의 결과를 예측하는 Branch Prediction과 같은 기술을 이용해 해결할 수 있습니다.


## 20. 인터럽트란?
    인터럽트는 입출력 장치 등의 외부적인 요인이나 예외 상황이 발생해 처리가 필요할 때 CPU에게 알려 발생한 상황을 처리하고 다시 실행 중인 작업으로 복귀하는 것을 의미합니다. 


### 하드웨어 인터럽트와 소프트웨어 인터럽트의 차이점은?
    차이점은 트리거 되는 소스에 있습니다. 하드웨어 인터럽트는 키보드나 네트워크 카드 같은 외부 장치에 의해 트리거되는 반면, 소프트웨어 인터럽트는 시스템 호출 등의 명령을 통해 프로그램에 의해 트리거됩니다.

### ISR(Interrupt Service Routine)이란?
    인터럽트가 발생할 때 실행되는 핸들러 함수입니다. 인터럽트가 발생했을 때 인터럽트를 처리하고 필요한 작업을 수행해 시스템이 이전 상태를 원활하게 재개할 수 있도록 보장합니다.

## 21. 가상 메모리란?
    디스크 공간을 RAM의 확장을 사용하는 메모리 관리 기술입니다. 이를 통해 시스템은 RAM과 디스크 공간 간의 데이터를 교환하면서 더 큰 애플리케이션을 처리할 수 있습니다.

## 22. 플래그 레지스터(Flag Register)란?
    플래그 레지스터는 산술 연산 결과의 상태를 나타내는 flag들이 모인 레지스터입니다.

# Stack

- 후입선출 LIFO(Last-In-First-Out) 자료구조
- 데이터를 차곡차곡 쌓아 올린 형태
- 가장 마지막에 삽입된 자료가 가장 먼저 삭제되는 구조

## Stack의 사용 사례

- 웹 브라우저 방문 기록 (뒤로 가기)
- 실행 취소
- 역순 문자열 만들기
- 후위 표기법 계산

<br>

## Java Collections - Stack

- Stack 클래스는 deprecated 되었다.
- Stack 클래스는 Vector 컬렉션을 상속받아 구현되었다.
- Vector 클래스는 자바 버전1부터 있었던 굉장히 오래된 클래스로, 취약점이 많다.
- 또한, 상속으로 인한 부모 메서드 공유 문제 때문에 사용자가 잘못되게 사용될 수 있다는 문제점이 존재한다.
- 즉, 자식 클래스에게는 부적합한 부모 클래스의 메소드가 상속되기 때문에 자식 클래스 인스턴스의 상태가 불안정해지게 된다.
- => Stack 클래스보다 **Deque 클래스를 사용할 것이 권장**된다. (자바 공식 문서)

## JVM Runtime Data Area(메모리 공간)의 Stack 영역

- 메소드 내에서 정의하는 기본 자료형에 해당되는 지역변수의 데이터 값이 저장되는 공간
- 메소드가 호출될 때 스택 영역에 스택 프레임이 생기고 그 안에 메소드를 호출
- primitive 타입의 데이터(int, boolean 등)에 해당되는 지역변수, 매개변수 데이터 값이 저장
- 메소드가 호출될 때 메모리에 할당되고 종료되면 메모리에서 사라진다.
- 후입선출 LIFO 특성ㅇ르 가지며, Scope의 범위를 벗어나면 스택 메모리에서 사라진다.

### 스택 프레임(Stack Frame)

- 하나의 메서드에 필요한 메모리 덩어리
- 하나의 메서드당 하나의 스택 프레임이 필요
- 메서드를 호출하기 직전 스택프레임을 자바 Stack에 생성한 후 메서드를 호출하게 된다.
